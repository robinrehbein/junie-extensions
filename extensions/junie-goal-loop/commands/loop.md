---
description: Self-judge loop — iterate toward a goal/criteria until you meet them or stall
---

Run a **self-judge loop** toward `<prompt|criteria>`.

1. Pick the success criteria: if an active goal exists in `<project>/.junie/goal.md`, use its
   `Success criteria`; otherwise distill explicit, checkable criteria from the argument and
   state them before starting. If you can't name any, ask — don't loop on vibes.
2. Loop, up to **8 iterations**: do one focused work pass → self-judge each criterion
   (met / not met, with evidence) → if nothing is left, stop and report; if a pass made zero
   progress, stop and report the blocker.
3. Keep the active goal's plan in sync — revise the checklist (`/goal` style) as you learn
   what the work actually needs.

See the goal-loop guideline for the full protocol (iteration cap, no-progress stop rule,
evidence requirement).
